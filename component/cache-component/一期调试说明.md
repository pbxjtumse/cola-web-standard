# cache-component 一期调试说明

## 当前版本定位

这是一期二级缓存 MVP：

- L1：Caffeine 本地缓存
- L2：Redis 分布式缓存
- Source：Demo 中使用 fake Repository 模拟数据库访问

缓存组件本身不负责数据库访问，数据库访问由业务 loader 提供。

## 启动步骤

```bash
cd component/cache-component
mvn clean install -DskipTests
```

启动 Redis：

```bash
docker run -d --name redis-cache-demo -p 6379:6379 redis:7.2
```

启动 demo：

```bash
cd cache-demo
mvn spring-boot:run
```

## 验证接口

第一次请求：

```bash
curl http://localhost:18081/demo/campaign-rules/10001/debug
```

预期：status=MISS_LOADED，level=SOURCE，日志出现 [SOURCE]。

第二次请求：

```bash
curl http://localhost:18081/demo/campaign-rules/10001/debug
```

预期：status=HIT，level=L1，一般不再出现 [SOURCE]。

查看源数据访问次数：

```bash
curl http://localhost:18081/demo/campaign-rules/source-load-count
```

删除缓存：

```bash
curl -X DELETE http://localhost:18081/demo/campaign-rules/10001/cache
```

查看 Redis key：

```bash
docker exec -it redis-cache-demo redis-cli keys 'cache:*'
```
