# 核心编排层 
它不直接绑定 Redis，也不直接绑定 Caffeine，而是负责缓存流程编排
# 核心职责
| 能力       | 说明                           |
| -------- | ---------------------------- |
| 缓存读流程编排  | 先查本地，再查 Redis，再执行 loader     |
| 缓存写流程编排  | 写 Redis、写本地、删除本地、延迟刷新等       |
| 缓存删除流程编排 | 单 key 删除、批量删除、按 namespace 删除 |
| key 构建   | 统一生成标准 key                   |
| TTL 计算   | 支持固定 TTL、随机 TTL、动态 TTL       |
| 空值缓存     | 防穿透                          |
| 加载互斥     | 防击穿                          |
| 热点保护     | 防止单 key 打爆                   |
| 异常处理     | Redis 异常时走降级策略               |
| 统计事件     | 记录 hit/miss/error/latency    |
| 策略路由     | 根据不同 cacheName 选择不同策略        |

# 本层目标
很多人做缓存组件失败，就是因为只封装了 RedisTemplate，没有做“流程编排”。
