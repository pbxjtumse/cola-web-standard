## 1. 当前一期目标

一期目标是本地 JVM 生产可用并行组件：

```text
统一任务提交
线程池治理
拒绝感知
timeout
fallback
cancel
状态追踪
错误分类
监听器
指标
Spring Boot 自动装配
单元测试覆盖
```

一期不做：

```text
跨 JVM 分布式取消
分布式锁
复杂重试补偿状态机
可视化控制台
```

## 2. 二期：动态线程池治理

能力：

```text
动态调整 corePoolSize / maxPoolSize / queueCapacity
配置中心接入
线程池快照持久化
线程池健康状态
自动告警
```

注意：动态调整不是简单 set 参数，需要考虑：

```text
配置合法性
调整失败回滚
正在运行任务不受影响
指标验证
```

## 3. 三期：重试与补偿组件

当前 `RetryPolicy` 可以作为元数据，但复杂重试建议独立组件。

原因：

```text
重试需要状态记录、次数、退避、幂等、死信、补偿、人工介入。
这些能力不应塞进并行组件主链路。
```

并行组件可以输出：

```text
AsyncError
RecoveryHint
TaskExecutionEvent
```

重试补偿组件消费这些事件。

## 4. 四期：分布式任务控制

能力：

```text
跨 JVM taskId 路由
远程 cancel(taskId)
任务所在节点登记
任务心跳
分布式任务快照
```

需要解决：

```text
任务在哪个节点
节点宕机怎么处理
取消命令如何投递
取消结果如何回传
```

## 5. 五期：控制台与可视化

能力：

```text
线程池列表
任务列表
任务状态流转
拒绝率、超时率、fallback 率
动态调整入口
告警配置
```

## 6. 与分布式锁组件的关系

并行组件解决的是：

```text
本地 JVM 内异步任务执行与治理。
```

分布式锁组件解决的是：

```text
跨 JVM / 跨节点资源互斥。
```

两者可以联动：

```text
任务执行前获取分布式锁
任务执行完成释放锁
任务超时或取消时释放锁
锁续约失败时取消任务
```

但不建议把分布式锁直接塞进并行组件一期主链路。它应该是独立组件，通过装饰器、模板或扩展点组合使用。
